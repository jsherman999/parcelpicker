Code overview

ParcelPicker

ParcelPicker is a local-first parcel intelligence app for Minnesota county property data. A browser UI accepts an address or map click, asks the Python backend to identify the seed parcel, optionally expands to touching parcel rings, stores the run in SQLite, and renders parcel geometry, owners, exports, and property-search links.

FastAPI Uvicorn SQLite WAL Leaflet map UI County ArcGIS REST APIs U.S. Census geocoder fallback Optional OpenAI/OpenRouter

What the app does

High-level behavior

The app runs locally at http://127.0.0.1:8091 and can also be exposed to LAN devices through HOST=0.0.0.0. It serves static HTML, CSS, and JavaScript, then exposes JSON APIs for parcel lookup, map-click lookup, run history, CSV export, and GeoJSON export.

The main workflow is: choose Wright, Hennepin, or St. Louis County; enter an address or click the map; choose rings 0, 1, or 2; then view the seed parcel, adjacent parcels, owners, source matches, map polygons, and external property links.

Runtime stack

Underlying services and software

  • No Redis, Celery, Postgres, Django, Flask, Node server, or npm build step is used.
  • Python backend uses FastAPI, uvicorn, pydantic, and httpx.
  • Configuration is loaded from .env through python-dotenv.
  • Persistent storage is a local SQLite file, defaulting to data/app.db, with WAL mode enabled.
  • Maps use Leaflet in the browser and OpenStreetMap tiles.
  • County data comes from public ArcGIS REST query endpoints.
  • Address fallback geocoding uses the U.S. Census oneline address geocoder.
  • Optional owner cleanup and run summaries use OpenAI or OpenRouter chat completions.
  • macOS background startup can be handled by launchd.

Call flow

Main parcel lookup path

Browser UI

backend/static/index.html loads Leaflet, styles.css, and app.js.

→

Lookup request

runLookup() posts /api/lookup; a map click posts /api/lookup/point.

FastAPI route

backend/main.py validates the request and selects a county runner with _get_runner().

→

Runner pipeline

ParcelLookupRunner checks the SQLite cache, creates a run, resolves the seed parcel, expands rings, and writes results.

Provider services

County adapters call ArcGIS. Address misses can fall back to Census geocoding and point-intersect parcel lookup.

→

Response and rendering

The API returns a run payload. app.js draws GeoJSON in Leaflet, fills metadata, table rows, exports, and property links.

API surface

HTTP routes

Route Method Handler path Purpose
/, /static/* GET backend/main.py Serves the browser app and static assets.
/api/health GET health() Basic health check returning {"status":"ok"}.
/api/providers/status GET provider_status() Reports supported county keys and whether the optional LLM helper is available.
/api/lookup POST lookup() Address-seeded parcel lookup with optional ring expansion and optional LLM assistance.
/api/lookup/point POST lookup_point() Map-click lookup by latitude and longitude, then the same ring-expansion pipeline.
/api/runs GET list_runs() Lists recent lookup runs from SQLite.
/api/runs/{run_id} GET get_run() Returns one stored run and all associated parcel rows.
/api/runs/{run_id}/csv GET get_run_csv() Downloads parcel rows as CSV.
/api/runs/{run_id}/geojson GET get_run_geojson() Downloads parcel geometries as a GeoJSON FeatureCollection.

Backend modules

What each major Python part does

File Responsibility
backend/main.py FastAPI app, route definitions, static file mounting, CORS, environment loading, service creation, and Uvicorn entry point.
backend/db.py SQLite schema, WAL setup, thread lock, run CRUD, parcel upsert, address aliases, cache lookup, cleanup, and run retrieval.
backend/services/base.py Shared ArcGIS query behavior: retry, throttle, request budget, address lookup, Census fallback, point lookup, adjacency, and geometry conversion.
backend/services/runner.py County-agnostic orchestration for address and point lookups, ring traversal, caching, owner normalization, summaries, and status handling.
backend/services/wright.py Wright County ArcGIS adapter: endpoint URL and field mapping for PID, OWNNAME, and PHYSADDR.
backend/services/hennepin.py Hennepin adapter using parcel layer plus address-points lookup to resolve addresses to PID.
backend/services/stlouis.py St. Louis County adapter with street-only address extraction and city/ZIP address reconstruction.
backend/services/registry.py County service factory and display labels.
backend/services/llm.py Optional OpenAI/OpenRouter chat client for owner-name normalization and short run summaries.

Core orchestration

ParcelLookupRunner responsibilities

  • run_lookup(): normalizes an address, tries a recent alias/run cache hit, then starts a fresh provider lookup.
  • run_lookup_from_point(): checks whether recent cached parcel polygons contain the clicked point before calling the county API.
  • _run_lookup_core(): creates the run, resolves the seed, traverses requested rings, writes parcels, generates summary, and completes or fails the run.
  • query_adjacent() calls use ArcGIS esriSpatialRelTouches to find parcels touching the current frontier.
  • Guardrails stop runaway work with MAX_PARCELS, MAX_REQUESTS_PER_RUN, and ADJACENT_LIMIT_PER_PARCEL.
  • Cache helpers reuse recent runs within RETENTION_DAYS and trim cached results down to the requested ring count.
  • LLM assistance is optional; deterministic uppercase owner normalization is always available as a fallback.

Data model

SQLite tables

lookup_runs

One row per lookup: input address or point label, rings, status, provider, LLM flag, seed parcel, summary, error, and timestamps.

parcels

Canonical parcel records keyed by parcel id: owner, normalized owner, site address, GeoJSON geometry, source, and update time.

run_parcels

Join table connecting runs to parcels, with ring number, seed flag, and match source such as address, Census point, or map click.

parcel_address_aliases

Normalized address-to-parcel cache for fast repeat address lookups during the retention window.

Frontend

Browser app breakdown

  • backend/static/index.html defines the county selector, address box, ring selector, LLM toggle, metadata panel, map, property links, and results table.
  • backend/static/styles.css styles the dark app shell, panels, controls, Leaflet map, link chips, table, and responsive layout.
  • backend/static/app.js holds county config, initializes Leaflet, calls backend APIs, renders parcel polygons, fills the table, and builds external links.
  • countyConfig controls map centers, placeholders, property-search links, ArcGIS JSON links, and Zillow/Realtor search suffixes per county.
  • renderRun() updates run metadata, export URLs, property links, map layers, and parcel table from the API response.
  • runLookupByPoint() sends clicked map coordinates to the backend and fills the address input with the detected seed address when available.

External calls

What talks to the network

Caller Service Used for Fallback behavior
BaseParcelService._query_county() County ArcGIS parcel query endpoints Address matching, point intersection, and touching-parcel adjacency. Retries transient failures, then fails the run with a provider error.
HennepinParcelService._query_address_points() Hennepin address-points ArcGIS layer Resolve address text to a Hennepin PID before loading the parcel geometry. Falls back to parcel-layer street and house-number matching.
BaseParcelService._geocode_with_census() U.S. Census oneline address geocoder Convert an address to coordinates when direct ArcGIS address lookup misses. Tries Census street-address match, then point-intersect; otherwise returns not found.
LLMService OpenAI or OpenRouter chat completions Optional owner-name normalization and concise run summaries. Disabled unless configured; deterministic owner normalization and summary remain in place.
app.js Leaflet tile layer from OpenStreetMap Browser map background tiles. The app still runs, but basemap tiles may not render.
app.js Zillow, Realtor, county property pages, ArcGIS JSON URLs External property links shown for the seed parcel. Links open in new tabs; lookup results are not dependent on them.

Configuration

Environment variables

  • HOST, PORT: bind address and port, defaulting to 0.0.0.0:8091 in the example env.
  • DB_PATH: SQLite database path.
  • RETENTION_DAYS: cache and cleanup window, default 30.
  • MAX_PARCELS, MAX_REQUESTS_PER_RUN, ADJACENT_LIMIT_PER_PARCEL: lookup expansion limits.
  • REQUEST_TIMEOUT_SECONDS, REQUEST_RETRIES, RETRY_BACKOFF_SECONDS, MIN_REQUEST_INTERVAL_SECONDS: provider request safeguards.
  • ENABLE_LLM_ASSIST, LLM_PROVIDER, LLM_MODEL, OPENAI_API_KEY, OPENROUTER_API_KEY: optional LLM behavior.
  • MAX_LLM_NORMALIZATIONS: cap on owner names sent to an LLM per run.
  • LOG_LEVEL: Python logging verbosity.

Operations

Run and deploy

  • Manual run: python -m backend.main.
  • Install dependencies with pip install -r requirements.txt.
  • backend/main.py loads .env, opens SQLite, creates all county services, creates one runner per county, and starts Uvicorn when run as a module.
  • SQLite data is created at runtime under data/app.db unless DB_PATH overrides it.
  • Provider calls are bounded by request budgets, retries, backoff, throttling, and parcel caps.
  • A macOS LaunchAgent can auto-start and keep the app alive after boot or crash.
  • Basic validation used in the project is python3 -m compileall backend.

Important architectural note

ParcelPicker is intentionally local and lightweight. There is no external job queue or separate database server: API requests do their work inline, external provider calls are async HTTP requests, and persistence is a single SQLite file protected by an in-process lock. SQLite uses WAL mode, while request budgets and parcel caps keep county ArcGIS lookups from growing beyond the configured limits.
